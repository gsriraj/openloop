# OpenLoop

[![CI](https://github.com/gsriraj/openloop/actions/workflows/ci.yml/badge.svg)](https://github.com/gsriraj/openloop/actions/workflows/ci.yml)
[![Crates.io](https://img.shields.io/crates/v/openloop)](https://crates.io/crates/openloop)
[![docs.rs](https://img.shields.io/docsrs/openloop)](https://docs.rs/openloop)
[![License: MIT](https://img.shields.io/badge/license-MIT-blue)](LICENSE)

**A CLI tool that delegates work to coding agents in a loop until a goal is achieved.**

Inspired by [Loop Engineering](https://addyosmani.com/blog/loop-engineering/) — designing systems that prompt agents instead of prompting them manually.

## How It Works

```
                     ┌─────────────────┐
                     │   Read GOAL.md   │
                     └────────┬────────┘
                              ▼
                     ┌─────────────────┐
                     │   Plan Next     │◄──── Ask agent:
                     │   Step (AI)     │      "What should we do next?"
                     └────────┬────────┘
                              ▼
                     ┌─────────────────┐
                     │   Dispatch to   │
                     │   Agent CLI     │
                     └────────┬────────┘
                              ▼
                     ┌─────────────────┐
                     │   Verify Goal   │◄──── Ask agent:
                     │   (AI Checker)  │      "Is the goal met?"
                     └────────┬────────┘
                              ▼
                     ┌─────────────────┐
                     │  Goal Met?      │────► Yes → Done
                     └────────┬────────┘
                              │ No → Loop again
```

## Installation

### From source

```bash
git clone https://github.com/gsriraj/openloop.git
cd openloop
cargo install --path .
```

### From crates.io

```bash
cargo install openloop
```

## Quick Start

Run the interactive setup wizard:

```bash
openloop
```

The wizard will:
1. Detect available agent CLIs on your `$PATH`
2. Help you define your project goal (with AI-guided refinement)
3. Let you select agents and models
4. Choose step-by-step or autopilot mode
5. Write config and start the loop

### Headless mode

```bash
# Single agent, autopilot
openloop --agent-cli opencode --model claude-sonnet-4-20250514 --autopilot

# Parallel with two agents
openloop --agent-cli opencode --agent-cli copilot --parallel --autopilot

# Custom config
openloop --agent-cli opencode --model-config temperature=0.3 --max-iterations 100

# Initialize a project
openloop --init
```

## CLI Reference

```
openloop                         # Interactive setup wizard (no args)
openloop [FLAGS] [OPTIONS]       # Headless mode with flags

Flags:
  --autopilot          Run fully autonomously (no human confirmations)
  --init               Scaffold .openloop/ config directory + example GOAL.md
  --status             Display current loop state and progress
  --parallel           Allow parallel agent execution (uses git worktrees)
  --help               Print help

Options:
  --goal <PATH>                 Goal file path (default: ./GOAL.md)
  --agent-cli <NAME>...         Agent CLI(s) to use (repeatable)
  --model <NAME>                Model override for all agents
  --model-config <KEY=VAL>...   Model params (repeatable)
  --max-iterations <N>          Max loop iterations (default: 50)
  --state-dir <PATH>            Config/state directory (default: .openloop)
```

## Configuration

Generated by the wizard at `.openloop/config.toml`:

```toml
goal = "GOAL.md"
max_iterations = 50
autopilot = false
parallel = false

[agents]
enabled = ["opencode"]

[agents.opencode]
model = "claude-sonnet-4-20250514"
model_config = { temperature = 0.7, verify_model = "gpt-4o" }

[state]
file = "state.md"
```
