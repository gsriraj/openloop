# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenLoop, please report it privately.

**Do not report security vulnerabilities through public GitHub issues.**

Instead, please email the maintainer directly or open a draft security advisory
on GitHub: https://github.com/gsriraj/openloop/security/advisories/new

You should receive a response within 48 hours. If not, please follow up.

## Scope

Security issues related to:

- Remote code execution via agent prompts
- Unauthorized file system access
- Exposure of sensitive configuration data

are considered in scope. General bugs should be reported through the issue tracker.
